​ 上一篇Spring Cloud文章说到了Spring Cloud Eureka（服务治理），知道eureka是如何将自己作为Eureka Server，以及其它Eureka Client注册到改Server上，这一篇文章要讲的是微服务之间是如何进行通讯的。在微服务架构中，业务都会被拆分成一个独立的服务，服务与服务的通讯是基于http restful的。Spring cloud有两种服务调用方式，一种是ribbon+restTemplate，另一种是feign。在这一篇文章首先讲解下基于ribbon+restTemplate。

Ribbon简介

​ Ribbon可以让我们轻松地将面向服务的REST模版请求自动转换成客户端负载均衡的服务调用。Ribbon虽然只是一个工具类框架，但是它几乎存在于每一个Spring Cloud构建的微服务和基础设施中。因为微服务间的调用，API网关的请求转发等内容，实际上都是通过Ribbon来实现的，以及经常使用的Feign也是封装了Ribbon来实现的工具。

客户端负载均衡

​ 负载均衡肯定都听过，但是我们日常中的负载均衡一般来自服务器端的负载均衡，这个一般通过Nginx网关通过一致性哈希算法或者其它进行服务器端的负载均衡。然后Spring Cloud中也有负载均衡，这个客户端负载均衡就是通过Ribbon完成。
​ Ribbon是支持负载均衡，默认的负载均衡策略是轮询，我们也是可以根据自己实际的需求自定义负载均衡策略的。只需继承AbstractLoadBalancerRule类，重写public Server choose(ILoadBalancer lb, Object key)即可。
